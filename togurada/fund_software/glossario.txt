API(Applicaction Programming interface) - Interface de programação de aplicativos:     1-Conjunto de rotinas e padrões de programação     2-Tem como objetivo acessar aplicativos de software     3-Plataformas baseados na web     4-Utilizada por programa  FRAMEWORK:     1-É um conjunto de bibliotecas     2-Utilizado para criar aplicações     3-Fornece uma estrutura de trabalho     4-Utilizado para facilitar o desenvolvimento de software     5-Um conjunto de códigos  IDE(Integrated Development Environment) - Ambiente de Desenvolvimento Integrado:     1-É um software     2-Utilizado para desenvolver programas de computador     3-Fornece ferramentas     4-Utilizado para facilitar a programação     5-Editor de texto     6-Integra diversas funcionalidades para desenvolvimento em uma interface gráfica  SDK(Software Development Kit) - Kit de Desenvolvimento de Software:     1-É um conjunto de ferramentas     2-Utilizado para criar aplicativos     3-Fornece bibliotecas     4-Utilizado para facilitar o desenvolvimento de software     5-Utilizado para criar aplicativos para plataformas específicas  Nativo:     1-Desenvolvido para funcionar em uma plataforma específica     2-Utilização de linguagens e ferramentas especificas para a plataforma em questão     Exemplo: Android, iOS, se usa Java e Kotlin para Android e Swift para iOS  Hibrido:     1-Implementação utiliza html, css e javascript     2-Desenvolvido para funcionar em várias plataformas     3-Utiliza frameworks como React Native, Flutter, Ionic  Serviços:     1-Processos de software.  Monolitico:     1-Roda com um unico processo.  Microserviços:     1-Abordagem de arquitetura de software que estrutura uma aplicação como uma coleção de serviços.     2-Cada serviço é um processo separado.     3-Comunicação entre os serviços é feita por meio de protocolos de comunicação.     4-Utiliza APIs para comunicação entre os serviços.  SOAP(Simple Object Access Protocol):     1-Protocolo de comunicação     2-Utilizado para troca de informações     3-Utilizado em serviços web     4-Utiliza XML     5-Utiliza HTTP  REST(Representational State Transfer):     1-Conjunto de restrições para criação de serviços web     2-Quando um serviço implementa esse padrão é chamado de RESTfull     3-Utiliza o protocolo HTTP     4-Utiliza JSON  Snapshot:     1-É uma imagem do estado de um projeto em um determinado momento     2-Utilizado para guardar o estado de um projeto     3-Utilizado para recuperar o estado de um projeto
