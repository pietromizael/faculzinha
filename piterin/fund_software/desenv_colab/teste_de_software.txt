Teste de software visa garantir que o software funcione corretamente, atendendo as especificações.

Error = Uma ação humana que produz um resultado incorreto.
Fault = Defeito no código que pode causar um erro.
Failure = Comportamento incorreto do software.

O custo para a correção de um bug aumenta exponencialmente conforme o tempo passa.

Estratégia de teste: Teste de unidade -> Teste de integração -> Teste de validação -> Teste de sistema.

Teste de unidade: Testa partes específicas do código, como funções e métodos.
Teste de integração: Testa a integração entre diferentes partes do sistema.
Teste de validação: Testa se o software atende aos requisitos do cliente.
Teste de sistema: Testa o sistema como um todo, incluindo integração com outros sistemas.

Depois de feito tudo isso, é feito a implantação do software em produção.
    1- Interpretadores
    2- Compiladores
    3- Depuradores
    4- Pacotes de software

Container é uma unidade de software que empacota e implanta uma aplicação e suas dependências.

A diferença entre contêiner e máquina virtual está relacionada basicamente com escala e portabilidade

Container empacota funções individuais em um único pacote, enquanto a máquina virtual empacota todo o sistema operacional.
